---

## 制作物について

- サイト名 / アプリ名: リンク（URL）。
- サービスの概要: 「誰の・どんな課題を解決するためのアプリか」。
- 担当範囲: 要件定義、UIデザイン、フロントエンド、バックエンドなど。
- 技術スタック: フロント、バック、DB、インフラで使用した言語やツール。
- 工夫した点・こだわった点: パフォーマンスの改善、セキュリティ対策、API設計など。
- 開発背景: 「なぜその技術を選定したのか」の理由。
- 構成図や資料がある場合は別ページに遷移し表示

## 資格について

- 取得資格については合格、取得として記載
- 勉強中取得予定のものについては学習中として記載
